## Base URL

```text
https://api.amarsia.com/v1
```

All endpoint paths below are appended to this base URL.

## Authentication

Send this header on every request:

```bash
x-api-key: YOUR_API_KEY
```

SDKs are coming soon. Right now the API is HTTP-first.

## Universal request pattern

Most requests use this JSON shape:

```json
{
  "content": [
    { "type": "text", "text": "Hello" }
  ],
  "variables": {
    "name": "value"
  }
}
```

| Field | Type | Required | Meaning |
|---|---|---|---|
| `content` | array | Yes | What you want the assistant to process |
| `variables` | object | No | Values injected into your assistant prompt |
| `meta` | object | No (conversation endpoints) | Labels for tracking/filtering conversations |

## Universal success pattern

Regular responses commonly include:

```json
{
  "content": "Assistant response",
  "model": "MODEL_NAME",
  "input_tokens": 100,
  "output_tokens": 80
}
```

Streaming responses send output chunks instead of one final object.

## `meta` in simple terms

`meta` is extra context tags you define (for example `userId`, `projectId`, `channel`).
It helps you filter and analyze conversations later.
