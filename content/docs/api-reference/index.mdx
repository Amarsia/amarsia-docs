## Base URL

```text
https://api.amarsia.com/v1
```

All endpoint paths below are appended to this base URL. Authenticated workflows require an `x-api-key` header. Public workflows can use allowlist-based access.

</TrackContent>

<TrackContent track="sdk">

## SDK package

Install: `npm install @amarsia/sdk`  &nbsp;·&nbsp; [npm ↗](https://www.npmjs.com/package/@amarsia/sdk)

Initialize once, use everywhere:

```ts
import { amarsia } from "@amarsia/sdk"

const client = amarsia.init({
  apiKey: process.env.AMARSIA_API_KEY!,
  deploymentId: process.env.AMARSIA_DEPLOYMENT_ID!,
})
```

</TrackContent>

## React package

Install: `npm install @amarsia/react @amarsia/sdk`  &nbsp;·&nbsp; [npm ↗](https://www.npmjs.com/package/@amarsia/react)

All hooks take the same `client` created by `amarsia.init(...)`:

```tsx
import { useRun, useStream, useConversation } from "@amarsia/react"
```

## Universal request shape

```json
{
  "content": [
    { "type": "text", "text": "Your input here" }
  ],
  "variables": {
    "VARIABLE_NAME": "value"
  }
}
```

| Field | Type | Required | Meaning |
|---|---|---|---|
| `content` | array | Yes | Input items — text, image, video, audio, url |
| `variables` | object | No | Values injected into assistant instructions |

`meta` is **not** universal. It is only valid when creating a new conversation (`POST /conversation` first message).

## Universal response shape

```json
{
  "content": "Assistant output...",
  "model": "model-name",
  "input_tokens": 42,
  "output_tokens": 61
}
```

Conversation responses also include `conversation_id` and may include timestamps (`created_at`, `updated_at`).
