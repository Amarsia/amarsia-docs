## What this does

Keeps multi-turn memory with conversation IDs. Each message in the same thread shares context with prior turns.

## Start conversation

<TrackContent track="sdk">

<ApiReferenceBlock
  title="conversation.run(options)"
  language="typescript"
  code={`const conversation = client.conversation
conversation.start()

await conversation.run({
  content: [{ type: "text", text: "Hi, I need help." }],
  meta: { userId: "user_123" },
})

console.log(conversation.id)    // "conv_abc123"
console.log(conversation.messages)`}
>

Call `conversation.start()` to begin a new thread. The ID is created on the first message.

| Option | Type | Required | Description |
|---|---|---|---|
| `content` | `MessageContent[]` | Yes | First message |
| `meta` | object | No | Custom labels for analytics/filtering |
| `variables` | object | No | Injected into assistant instructions |
| `historyLimit` | number | No | Max past messages to include as context |

</ApiReferenceBlock>

</TrackContent>

<TrackContent track="react">

<ApiReferenceBlock
  title="useConversation(client)"
  language="tsx"
  code={`const { conversation, id, messages, status } = useConversation(client)

await conversation.stream({
  content: [{ type: "text", text: "Start support chat." }],
})

console.log(id)       // "conv_abc123"
console.log(messages) // [{ role: "user", ... }, { role: "assistant", ... }]`}
>

`useConversation(client)` manages thread state reactively. `messages` updates after each turn.

| Return | Meaning |
|---|---|
| `conversation` | Conversation controller (same API as SDK) |
| `id` | Current conversation ID (null before first message) |
| `messages` | Reactive array of all turns |
| `status` | `idle \| loading \| streaming \| success \| error` |

</ApiReferenceBlock>

</TrackContent>

<TrackContent track="rest-api">

<ApiReferenceBlock
  method="POST"
  path="/v1/runner/{deployment_id}/conversation"
  language="bash"
  code={`curl https://api.amarsia.com/v1/runner/YOUR_DEPLOYMENT_ID/conversation \\
  -H "Content-Type: application/json" \\
  -H "x-api-key: YOUR_API_KEY" \\
  -d '{
    "content": [{ "type": "text", "text": "Hi, I need help." }],
    "meta": { "userId": "user_123" }
  }'`}
>

Creates a new conversation and returns `conversation_id`. Save it — every follow-up message needs it.

</ApiReferenceBlock>

</TrackContent>

## Continue conversation

<TrackContent track="sdk">

<ApiReferenceBlock
  title="conversation.stream(options)"
  language="typescript"
  code={`await client.conversation.stream({
  content: [{ type: "text", text: "Continue this chat." }],
  historyLimit: 10,
})`}
>

The SDK tracks the ID automatically. Just keep calling `conversation.run(...)` or `conversation.stream(...)` in the same thread.

</ApiReferenceBlock>

</TrackContent>

<TrackContent track="react">

<ApiReferenceBlock
  title="conversation.stream(options)"
  language="tsx"
  code={`await conversation.stream({
  content: [{ type: "text", text: "Please summarize in 2 bullets." }],
  historyLimit: 10,
})`}
>

The hook tracks the thread ID for you. `messages` and `status` update reactively after each turn.

</ApiReferenceBlock>

</TrackContent>

<TrackContent track="rest-api">

<ApiReferenceBlock
  method="POST"
  path="/v1/runner/{deployment_id}/conversation/stream"
  language="bash"
  code={`curl -N https://api.amarsia.com/v1/runner/YOUR_DEPLOYMENT_ID/conversation/stream \\
  -H "Content-Type: application/json" \\
  -H "x-api-key: YOUR_API_KEY" \\
  -d '{
    "conversation_id": "conv_abc123",
    "content": [{ "type": "text", "text": "Continue this chat." }],
    "history_limit": 10
  }'`}
>

`conversation_id` is required on every follow-up. Without it, a new thread starts.

| Body field | Type | Required |
|---|---|---|
| `conversation_id` | string | Yes (continuation) |
| `content` | array | Yes |
| `history_limit` | number | No |

</ApiReferenceBlock>

</TrackContent>

## Fetch history

- Messages: `GET /v1/runner/conversation/{conversation_id}/messages?page=1&page_size=20`
- All threads: `GET /v1/runner/conversations?page=1&page_size=20`

## `meta` — tagging conversations

Attach custom labels for filtering and analytics:

```json
{
  "meta": {
    "userId": "user_123",
    "projectId": "project_456",
    "source": "web-chat"
  }
}
```
