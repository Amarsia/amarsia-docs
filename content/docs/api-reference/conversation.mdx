---
title: Conversation API
description: Endpoint and SDK reference for stateful multi-turn conversations.
---

Conversations keep memory across messages using a `conversation_id`. Keep that ID if you want to continue the same thread later.

`content` in conversation endpoints uses the same `MessageContent[]` schema as Runner API.
See [Runner API — content body structure](/api-reference/runner#content-body-structure-messagecontent) and [Multimodal](/client-usage/multimodal).

<TrackContent track="rest-api">

**Endpoints at a glance**

| Method | Path | Purpose |
|---|---|---|
| `POST` | `/v1/runner/{deployment_id}/conversation` | Start or continue a conversation (non-stream) |
| `POST` | `/v1/runner/{deployment_id}/conversation/stream` | Continue a conversation with streaming |
| `GET` | `/v1/runner/conversation/{conversation_id}/messages` | Read paginated message history |
| `GET` | `/v1/runner/{deployment_id}/conversations` | List conversations (supports `meta` filters) |

**`POST /v1/runner/{deployment_id}/conversation`**

Used for both first message and follow-up messages.

<TrackContent track="rest-api">

<ApiReferenceBlock
  method="POST"
  path="/v1/runner/{deployment_id}/conversation"
  language="bash"
  code={`curl https://api.amarsia.com/v1/runner/YOUR_DEPLOYMENT_ID/conversation \\
  -H "Content-Type: application/json" \\
  -H "x-api-key: YOUR_API_KEY" \\
  -d '{
    "content": [{ "type": "text", "text": "Hi, I need help with my order." }],
    "variables": { "CUSTOMER_TIER": "pro" },
    "meta": { "userId": "user_123", "channel": "web-chat" }
  }'`}
>

| Body field | Type | Required | Notes |
|---|---|---|---|
| `content` | `MessageContent[]` | Yes | Input parts (`text`, `image`, `video`, `audio`, `url`) |
| `conversation_id` | string | No | Required only for continuation |
| `variables` | object | No | First message only |
| `meta` | object | No | First message only |
| `history_limit` | number | No | Continuation only |

</ApiReferenceBlock>

</TrackContent>

Response shape:

```json
{
  "conversation_id": "conv_abc123",
  "content": "Of course. What's your order number?",
  "model": "MODEL_NAME",
  "input_tokens": 42,
  "output_tokens": 61,
  "request_id": "req_123",
  "created_at": "2026-04-20T10:00:00Z",
  "updated_at": "2026-04-20T10:00:00Z"
}
```

| Response field | Type | Notes |
|---|---|---|
| `conversation_id` | string | Save this to continue later |
| `content` | string or object | Assistant output |
| `model` | string | Returned when available |
| `input_tokens` | number | Returned when available |
| `output_tokens` | number | Returned when available |
| `request_id` | string | Returned when available |
| `created_at` | string | Returned when available |
| `updated_at` | string | Returned when available |

**`POST /v1/runner/{deployment_id}/conversation/stream`**

Streaming continuation endpoint.

<TrackContent track="rest-api">

<ApiReferenceBlock
  method="POST"
  path="/v1/runner/{deployment_id}/conversation/stream"
  language="bash"
  code={`curl -N https://api.amarsia.com/v1/runner/YOUR_DEPLOYMENT_ID/conversation/stream \\
  -H "Content-Type: application/json" \\
  -H "x-api-key: YOUR_API_KEY" \\
  -d '{
    "conversation_id": "conv_abc123",
    "content": [{ "type": "text", "text": "Summarize in 2 bullets." }],
    "history_limit": 10
  }'`}
>

| Body field | Type | Required |
|---|---|---|
| `conversation_id` | string | Yes |
| `content` | `MessageContent[]` | Yes |
| `history_limit` | number | No |

Use `-N` with curl so chunks are not buffered.

</ApiReferenceBlock>

</TrackContent>

**`GET /v1/runner/conversation/{conversation_id}/messages`**

Returns paginated messages for one conversation.

| Query param | Type | Default |
|---|---|---|
| `page` | number | `1` |
| `page_size` | number | `20` |

Response shape:

```json
{
  "messages": [{ "id": "msg_1", "role": "user", "content": [{ "type": "text", "text": "Hi" }], "created_at": "..." }],
  "page": 1,
  "page_size": 20,
  "total": 1,
  "has_more": false
}
```

**`GET /v1/runner/{deployment_id}/conversations`**

Lists conversations with pagination. You can filter by `meta` fields using query params.

Example:

```text
GET /v1/runner/{deployment_id}/conversations?page=1&page_size=20&userId=user_123&channel=web-chat
```

Response shape:

```json
{
  "conversations": [{ "conversation_id": "conv_abc123", "meta": { "userId": "user_123" } }],
  "page": 1,
  "page_size": 20,
  "total": 1,
  "has_more": false
}
```

</TrackContent>

## SDK surface

<TrackContent track="sdk">

<ApiReferenceBlock
  title="conversation.start(conversationId?, deploymentId?)"
  language="typescript"
  code={`const conversation = client.conversation

conversation.start() // new local context
conversation.start("conv_existing_123") // resume existing conversation
`}
>

`start(...)` sets active conversation context (`conversation.id` + `deploymentId`) and clears local conversation state.
Use `start("conv_...")` to resume a saved conversation.
Do not switch deployment in the middle of a thread and expect the same conversation continuity.
For most apps, if you need a different deployment, initialize a new client with that deployment id.
The second `deploymentId` argument is an advanced option and is rarely needed.

</ApiReferenceBlock>

<ApiReferenceBlock
  title="conversation.run(options) / conversation.stream(options)"
  language="typescript"
  code={`await conversation.run({
  content: [{ type: "text", text: "Hello" }],
  variables: { CUSTOMER_TIER: "pro" }, // first message only
  meta: { userId: "user_123" }, // first message only
  historyLimit: 10, // continuation only
})`}
>

| Option | Type | Required | Notes |
|---|---|---|---|
| `content` | `MessageContent[]` | Yes | Must be non-empty |
| `variables` | object | No | First message only |
| `meta` | object | No | First message only |
| `historyLimit` | number | No | Continuation only |
| `signal` | `AbortSignal` | No | Cancel in-flight request |

</ApiReferenceBlock>

<ApiReferenceBlock
  title="conversation.loadMessages(options)"
  language="typescript"
  code={`await conversation.loadMessages() // uses active conversation.id

await conversation.loadMessages({
  conversationId: "conv_existing_123",
  page: 1,
  pageSize: 20,
  append: true,
})`}
>

| Option | Type | Required | Notes |
|---|---|---|---|
| `conversationId` | string | No | Uses active `conversation.id` if omitted |
| `page` | number | No | Defaults to API default |
| `pageSize` | number | No | Defaults to API default |
| `append` | boolean | No | Merge into existing `conversation.messages` and dedupe by message id |

</ApiReferenceBlock>

<ApiReferenceBlock
  title="conversation.list(options)"
  language="typescript"
  code={`await conversation.list({
  page: 1,
  pageSize: 20,
  meta: { userId: "user_123", channel: "web-chat" },
})`}
>

| Option | Type | Required |
|---|---|---|
| `page` | number | No |
| `pageSize` | number | No |
| `meta` | object | No |

</ApiReferenceBlock>

`conversation.abort()` aborts any in-flight `conversation.run(...)` or `conversation.stream(...)` call.

State fields exposed by `client.conversation`:
- `id`, `deploymentId`, `status`, `data`, `live`, `error`, `meta`, `raw`
- `messages`, `messagesPageInfo`, `conversations`, `conversationsPageInfo`

</TrackContent>

## React surface

<TrackContent track="react">

<ApiReferenceBlock
  title="useConversation(client)"
  language="tsx"
  code={`const { conversation, id, status, live, data, messages } = useConversation(client)

// resume from stored id
useEffect(() => {
  const stored = "conv_abc123"
  if (stored) conversation.start(stored)
}, [conversation])`}
>

`useConversation(client)` returns reactive conversation state plus the same conversation controller methods from the SDK.

</ApiReferenceBlock>

</TrackContent>

## Error notes

- `content` must be a non-empty array.
- `variables` and `meta` are rejected on continuation calls.
- See [Errors reference](/api-reference/errors) for status and SDK error mapping.
