## One rule

Every API request must include:

```bash
x-api-key: YOUR_API_KEY
```

If this header is missing or invalid, request fails.

## Where to get keys

Create or revoke keys in:

- [API Keys dashboard](https://amarsia.com/dash/api_keys)

## Simple request example

```bash
curl https://api.amarsia.com/v1/runner/YOUR_DEPLOYMENT_ID \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{
    "content": [
      { "type": "text", "text": "Hello" }
    ]
  }'
```

## Common auth errors

| Status | Meaning | Fix |
|---|---|---|
| `401` | Missing or invalid key | Pass a valid key |
| `403` | Key is valid but not allowed | Check project access/permissions |

## Security basics

- Keep keys on server-side only
- Never commit keys to git
- Rotate key immediately if leaked
