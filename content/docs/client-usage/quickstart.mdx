## What you need

Two things before you write any code:

**API Key** — your secret credential for authenticating requests.

Get it from the [Amarsia dashboard](https://amarsia.com/dash/api_keys) under **API Keys**. Store it as an environment variable — never hardcode it.

```bash
AMARSIA_API_KEY=your_key_here
```

**Deployment ID** — identifies which assistant to run. In Amarsia, you build and deploy assistants. Each deployed assistant has its own Deployment ID. Find it in the assistant's **Deploy** tab in the dashboard.

```bash
AMARSIA_DEPLOYMENT_ID=dep_abc123
```

## Install

<TrackContent track="sdk">

```bash
npm install @amarsia/sdk
```

</TrackContent>

<TrackContent track="react">

```bash
npm install @amarsia/react @amarsia/sdk
```

Both packages are needed. `@amarsia/sdk` is the core client, `@amarsia/react` wraps it with hooks.

</TrackContent>

<TrackContent track="rest-api">

No package needed. Use any HTTP client — `curl`, `fetch`, `axios`, whatever you have.

Base URL for all requests:

```text
https://api.amarsia.com/v1
```

</TrackContent>

## Initialize the client

<TrackContent track="sdk">

Create the client once and reuse it everywhere:

```ts
import { amarsia } from "@amarsia/sdk"

const client = amarsia.init({
  apiKey: process.env.AMARSIA_API_KEY!,
  deploymentId: process.env.AMARSIA_DEPLOYMENT_ID!,
})

export default client
```

</TrackContent>

<TrackContent track="react">

Create the client in a shared file, then import it into your components:

```ts
// lib/amarsia.ts
import { amarsia } from "@amarsia/sdk"

const client = amarsia.init({
  apiKey: process.env.NEXT_PUBLIC_AMARSIA_API_KEY!,
  deploymentId: process.env.NEXT_PUBLIC_AMARSIA_DEPLOYMENT_ID!,
})

export default client
```

> **Note:** Use `NEXT_PUBLIC_` prefix only if the key is safe to expose client-side. For sensitive keys, call Amarsia from a server action or API route instead.

</TrackContent>

<TrackContent track="rest-api">

No initialization needed. Just include these headers on every request:

```bash
Content-Type: application/json
x-api-key: YOUR_API_KEY
```

</TrackContent>

## Variables

Your assistant's system instructions can have dynamic slots called **variables**. They let you inject real data into the prompt at runtime — things like a user's name, account info, or any context the assistant needs.

**Where to find them:** Open your assistant in the dashboard and check the system instructions. Variables are always written in `UPPER_CASE_WITH_UNDERSCORES` like `CLIENT_EMAIL` or `PRODUCT_NAME`.

**How to pass them:**

<TrackContent track="sdk">

```ts
const result = await client.run({
  content: [{ type: "text", text: "Help this customer." }],
  variables: {
    CLIENT_EMAIL: "jane@example.com",
    PLAN_NAME: "Pro",
  },
})
```

</TrackContent>

<TrackContent track="react">

```tsx
await run({
  content: [{ type: "text", text: "Help this customer." }],
  variables: {
    CLIENT_EMAIL: "jane@example.com",
    PLAN_NAME: "Pro",
  },
})
```

</TrackContent>

<TrackContent track="rest-api">

```json
{
  "content": [{ "type": "text", "text": "Help this customer." }],
  "variables": {
    "CLIENT_EMAIL": "jane@example.com",
    "PLAN_NAME": "Pro"
  }
}
```

</TrackContent>

The variable names must match exactly what's in your assistant's instructions. If there's a mismatch, the slot will be left blank.

## Your first request

<TrackContent track="sdk">

```ts
import client from "./lib/amarsia"

const result = await client.run({
  content: [{ type: "text", text: "Say hello." }],
})

console.log(result.content)
```

</TrackContent>

<TrackContent track="react">

```tsx
"use client"
import { useRun } from "@amarsia/react"
import client from "@/lib/amarsia"

export function HelloButton() {
  const { run, status, data } = useRun(client)

  return (
    <div>
      <button onClick={() => void run({ content: [{ type: "text", text: "Say hello." }] })}>
        Go
      </button>
      {status === "loading" && <p>Thinking...</p>}
      {data && <p>{String(data.content)}</p>}
    </div>
  )
}
```

</TrackContent>

<TrackContent track="rest-api">

```bash
curl https://api.amarsia.com/v1/runner/YOUR_DEPLOYMENT_ID \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{ "content": [{ "type": "text", "text": "Say hello." }] }'
```

</TrackContent>
