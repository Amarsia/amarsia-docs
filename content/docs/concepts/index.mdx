## Why read this first?

The concepts section explains _how_ Amarsia thinks about problems. Understanding these abstractions makes every other part of the docs — features, integrations, and the API — easier to work with.

## Core abstractions

| Concept | What it is |
|---|---|
| **Workspace** | The top-level container for all your data and configuration |
| **Resource** | Any entity managed by the platform (users, records, events, etc.) |
| **Event** | An immutable record of something that happened |
| **Integration** | A connection to a third-party service |
