## Overview

Amarsia is built around eight concepts that cover the full lifecycle of an assistant: building its behavior, grounding it in your own knowledge, giving it real-world tools, proving it works before you ship, and securing and monitoring it in production.

The goal is practical: less engineering overhead, fewer blind spots, and faster iteration when your product needs change.

## Outcome-driven view

| What you need | Concept that solves it |
|---|---|
| Reliable AI behavior you can update without app rewrites | [Assistants](/concepts/assistants) |
| Better factual responses from your own content | [Knowledge base](/concepts/knowledge-base) |
| Real-world actions like API calls, database queries, and notifications | [Actions](/concepts/actions) |
| Confidence that changes didn't break existing behavior | [Test cases](/concepts/test-cases) |
| Measurable quality scores for every production run | [Evaluations](/concepts/evaluations) |
| Visibility into failures, reasoning, and tool runs | [Usage](/concepts/usage) |
| Secure access from your product or backend | [API key](/concepts/api-key) |
| Safe public access from browser apps you own | [Security](/concepts/security) |
