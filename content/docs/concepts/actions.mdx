## Overview

Actions are tools your assistant can execute, such as API calls, database queries, messages, and CRM operations. Every action is deployed in an isolated cloud environment and can be assigned selectively per assistant.

Selective tool assignment matters: giving an assistant only the tools it needs reduces error-prone tool selection and improves output reliability.

## How actions work

1. Create an action from a template or custom code.
2. Define parameters and secrets.
3. Test and publish the action.
4. Attach selected actions to an assistant.
5. Review action-specific logs and iterate.

## Templates and categories

Amarsia includes ready templates across:

- REST API
- Database
- Communication
- Commerce
- CRM and operations

## MCP import

You can connect external or self-hosted MCP servers and import discovered tools as actions.

- Test connection first.
- Review discovered tools.
- Save to create actions in draft mode.
- Refresh later to sync newly available tools.
